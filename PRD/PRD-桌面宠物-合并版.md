## 0. 关键产品决策（已拍板）

| 编号 | 决策点 | 结论 |
|---|---|---|
| A | 防沉迷触发方式 | **基于"连续使用时长"触发**（非固定时间点） |
| B | 防沉迷是否强制 | **不强制，定位为"善意提醒"**，休息期间允许随时关闭程序 |
| C | 全屏独占场景 | 用户玩独占全屏游戏/视频时，防沉迷及提醒**允许不弹出**，不做底层强制穿透 |

| F | 素材格式管控 | **不限制格式、不做处理**，由用户自行负责观感（见 §7 风险提示） |
| G | 提示音 | **内置若干音效 + 支持用户上传自定义音频** |

## 1. 执行摘要（Executive Summary）

### 1.1 问题陈述
长时间使用电脑办公/学习的用户缺少一个低打扰、有情感陪伴感的桌面伙伴，同时也缺少与之一体化的"事项提醒"和"基于真实使用时长的健康休息提醒"——现有提醒类工具冰冷、易被忽略，纯桌面宠物又缺乏实用价值。

### 1.2 解决方案
一款常驻桌面、悬浮于所有窗口之上、可拖动的动态虚拟宠物。以"图层叠加"模型统一承载三大功能——**陪伴办公**（基础层）、**防沉迷善意提醒**（覆盖层）、**事项提醒**（最高叠加层），通过控制面板配置素材、时间、文案与提示音，全本地运行、配置持久化。

### 1.3 设计目标
- 形象动态、自然，悬浮观感无明显边框（依赖透明素材）。
- 交互直觉、不易误操作（尤其避免误退出）。
- 三功能按优先级有序协同，状态切换逻辑清晰、无冲突。

### 1.4 成功标准（可量化 KPI）

| # | 指标 | 目标值 | 验证方式 |
|---|---|---|---|
| K1 | 空闲态资源占用 | 宠物悬浮 + 视频常驻播放时，CPU 平均 ≤ 5%、内存 ≤ 200MB（1080p 以内素材） | 任务管理器 / `process.getProcessMemoryInfo` 采样 1 小时 |
| K2 | 稳定性（崩溃率） | 连续运行 72 小时无崩溃、无内存泄漏（内存增长 ≤ 10%） | 长稳测试 + 内存曲线监控 |
| K3 | 交互准确率 | 单击/拖动判定准确率 100%（移动阈值生效，无误触退出） | 手测用例集（≥ 30 次拖动 / 单击混合操作） |
| K4 | 提醒可靠性 | 设定提醒到点触发误差 ≤ 1 秒；重复规则（每天/每周/工作日）100% 正确 | 自动化时间用例 |
| K5 | 防沉迷计时正确性 | 连续使用满阈值触发、空闲满阈值重置、被提醒打断暂停/恢复，逻辑 100% 正确 | 状态机用例覆盖 §4.3 全部规则 |
| K6 | 场景鲁棒性 | 锁屏/解锁、睡眠/唤醒、显示器插拔下不崩溃、状态正确恢复 | 场景回归用例 |

## 2. 用户画像与用户故事

### 2.1 用户画像（Personas）
- **P1 · 长时办公族（核心）**：每天连续用电脑 6h+，需要轻量陪伴 + 健康休息提醒，但反感强制锁屏类工具。重视"不打扰、可随时关"。
- **P2 · 学生/自习党**：用番茄式节奏学习，需要事项提醒（交作业、上网课）+ 防沉迷，喜欢可爱形象与个性化素材。
- **P3 · 个性化玩家**：把桌宠当桌面装饰，重度使用自定义素材/文案/音效，关注观感与缩放自由度。

### 2.2 用户故事与验收标准（User Stories + AC）

**US-1｜陪伴办公（默认态）** → 详见 §4.1, §6.1
> 作为长时办公族，我想要一只动态宠物悬浮在桌面上，以便工作时获得陪伴感而不被打扰。
- 启动后默认进入陪伴办公态，宠物 always-on-top 悬浮于普通窗口之上。
- 左键拖动可移动，超过移动阈值才判定为拖动；松手后位置持久化，重启恢复。
- 鼠标悬停：若该素材绑定了文案（≤20字）则头顶显示，移开消失；未配置则不显示。
- 多素材时按设定时间（5–60 分钟）轮播；单素材时不可设时间。
- 锁屏时宠物休眠（暂停动画与计时），解锁恢复。

**US-2｜事项提醒** → 详见 §4.2, §6.2
> 作为学生，我想要在指定时间被宠物提醒，以便不错过重要事项。
- 到点时宠物切换为提醒素材，上方弹出提醒面板（白底黑字 + "我知道了"）。
- 面板**不点不消失**；多条同时到点以可堆叠列表呈现，逐条独立确认。
- 点"我知道了"后该提醒记为完成，面板消失，宠物恢复底层状态。
- 支持重复规则：单次 / 每天 / 每周 / 工作日。
- 提示音可开关，支持内置音效或自定义上传（决策 G）。
- 时间选择遵循 §8：分 0–59、时 0–23、日按年月动态、禁止选过去时间。

**US-3｜防沉迷善意提醒** → 详见 §4.3, §6.3
> 作为长时办公族，我想要在连续使用过久时被温和提醒休息，以便保护健康，且能随时关闭。
- 基于鼠标/键盘活动检测，**连续使用满阈值（默认 90 分钟）**触发，全屏展示休息素材。
- 无操作满空闲阈值（默认 10 分钟）/ 锁屏 / 睡眠 → 重置累计计时。
- 达到设定"休息时长"后自动结束，恢复陪伴办公态。
- **善意提醒，用户可随时手动关闭/退出程序，不阻止关闭**（决策 B）。
- 独占全屏场景下可延迟/不弹出，退出全屏后再处理（决策 C）。

**US-4｜图层协同与打断处理** → 详见 §3
> 作为用户，我想要三功能互不冲突，以便体验顺畅、无歧义。
- 优先级 事项提醒 > 防沉迷 > 陪伴办公；提醒为最高叠加层，叠加在防沉迷画面之上；防沉迷倒计时在被提醒打断期间暂停，确认后继续；暂停宠物时提醒与防沉迷计时一并暂停。

**US-5｜控制面板配置与持久化** → 详见 §6, §9
> 作为个性化玩家，我想要统一面板配置素材/时间/文案/音效，以便自由定制。
- 双击宠物或右键菜单打开面板；三分区 + 通用项（开机自启、提示音开关、状态显示/暂停）；所有配置本地持久化、重启恢复；素材自由缩放（最大全屏，最小快捷图标大小）。

**US-6｜防误操作交互** → 详见 §2.4
> 作为用户，我想要不会误退出宠物，以便放心使用。
- 退出仅经右键菜单的"退出"，单击不退出；右键菜单含 打开面板 / 暂停宠物 / 退出；双击打开面板。

### 2.3 非目标（Non-Goals）
- ❌ **不接入 AI 对话/语音**（本期明确排除）。
- ❌ **不做强制防沉迷**：不锁屏、不阻止关闭、不做底层全屏强制穿透（决策 B/C）。
- ❌ **不做素材自动透明化/裁剪/转码处理**：观感由用户负责（决策 F）。
- ❌ **不限制上传素材格式**（仅风险提示，不拦截）（决策 F）。
- ❌ **不做多显示器自选屏配置**（本期默认显示在宠物所在屏，后续按需扩展）。
- ❌ **不做云同步 / 账号体系 / 联网功能**：纯本地。
- ❌ **不做移动端 / macOS / Linux**：本期仅 Windows 桌面。

## 2.4 鼠标交互定义

> 修正原因：原方案"右键单击直接退出"易误触；"双击进面板"会引入单击延迟。统一收敛到右键菜单。

| 操作 | 行为 |
|---|---|
| 左键按住拖动 | 移动宠物（需超过移动阈值才判定为拖动，避免与单击冲突） |
| 鼠标悬停（hover） | 触发宠物互动反应；陪伴办公模式下若配置了文案则在头顶显示文案 |
| 左键单击 | 触发一次宠物互动反应 |
| 左键双击 | 打开控制面板 |
| **右键单击** | **弹出右键菜单**：打开控制面板 / 暂停宠物 / 退出 |
| 退出程序 | 仅通过右键菜单中的"退出"，不再由单击直接触发 |

> 术语统一：操作系统层面不存在"轻触"事件，原文"轻触（非点击）"= 鼠标悬停（hover/mouseover），全文统一为"悬停"。

## 3. 功能优先级与状态机

### 3.1 优先级
**事项提醒 > 防沉迷 > 陪伴办公**

### 3.2 分层模型（图层叠加）
> 修正原因：原"先提醒→再防沉迷→再陪伴"的串行替换模型，会导致防沉迷被提醒无限期挂起、计时归属不明。

- **基础层（常驻）**：陪伴办公。打开程序默认进入；若未配置提醒/防沉迷，则一直处于此状态。
- **防沉迷层（覆盖层）**：基于使用时长触发，全屏动画覆盖基础层；倒计时结束后回到基础层。
- **事项提醒层（最高叠加层）**：以面板形式叠加在任意状态之上，不替换底层状态。

### 3.3 关键交互规则
1. 防沉迷处于活动状态时，若事项提醒到点 → 提醒面板**叠加显示在防沉迷画面之上**。
2. 防沉迷的"关闭倒计时"在被提醒打断期间**暂停**，提醒处理完（用户点"我知道了"）后**继续计时**。
3. 提醒面板不点确认不消失；多条提醒到点时以**可堆叠列表**呈现，每条独立确认。
4. 防沉迷为善意提醒，**用户可随时关闭/退出程序**（决策 B），不阻止关闭。
5. 暂停宠物时，提醒与防沉迷计时**一并暂停**。

## 4. 功能详述

### 4.1 陪伴办公（默认模式）
- 宠物保持动态展示。
- 鼠标悬停宠物：若配置了绑定文案则在头顶显示，移开消失；未配置则无文案。
- 电脑锁屏：宠物休眠；解锁恢复。
- 多素材按设定时间轮播（见 §6.1）。

### 4.2 事项提醒
- 到达设定时间点 → 宠物形象切换为该模块上传的素材，并在宠物上方弹出**提醒面板**。
- 提醒面板：白底黑字文案 + "我知道了"按钮。
- 用户点击"我知道了" → 该提醒记为已完成，面板消失，宠物**恢复到底层状态**（防沉迷或陪伴办公）。
- 不点击则面板持续显示，不自动消失。
- **提示音**（决策 G）：到点时可选播放提示音；提供内置音效，也支持用户上传自定义音频。

### 4.3 防沉迷（善意提醒）
- **触发条件**：连续使用电脑达到设定时长后触发（基于鼠标/键盘活动检测）。
  - 锁屏 / 睡眠 / 无操作达到设定空闲阈值 → **重置累计使用计时**。
- 触发后：全屏展示该模块上传的素材，提示休息。
- **默认参数**（决策 E）：连续使用 90 分钟触发；无操作满 10 分钟重置累计计时。两者均可在控制面板修改。
- **关闭方式**：达到"休息时长"后自动消失，宠物恢复陪伴办公状态；用户也可随时手动关闭/退出（决策 B）。
- 独占全屏场景下可不弹出（决策 C）。

## 5. 宠物展示形态与系统适配

### 5.1 基本形态
- 宠物常驻桌面，**悬浮于所有普通窗口之上**（always-on-top）。
- 支持**鼠标左键拖动**移动位置，拖动结束后**记忆位置**（重启后恢复）。
- 素材为动态形象（视频 / 动图），见 §7 文件规范。

### 5.2 显示与系统适配
- **多显示器**：宠物显示在最后停留的显示器上；全屏类功能（防沉迷）默认覆盖宠物所在的那块屏。
- **独占全屏场景**（决策 C）：检测到前台为独占全屏程序时，防沉迷与提醒可延迟/不弹出，待退出全屏后再处理。
- **锁屏/睡眠**：进入锁屏时宠物休眠（暂停动画与计时），解锁后恢复。

## 6. 控制面板

面板分三个功能分区：陪伴办公、事项提醒、防沉迷。各区通用设置含：素材上传、时间设置、文案设置；另有各区专项设置。
通用项：开机自启开关、配置持久化、提示音开关、宠物当前状态显示/暂停。

### 6.1 陪伴办公分区
- **素材上传**：支持视频 / 动图；**可上传多个**；支持自由缩放，最大=全屏，最小=普通程序快捷图标大小。
- **时间设置**：仅在多素材时可设；单位分钟，**最小 5 分钟，最多 60 分钟**；到时切换展示已上传素材；仅 1 个素材时不可设置时间。
- **文案设置**：
  - 字数 ≤ 20 字。
  - 支持与素材**一对一绑定**（一个素材一条专属文案）。
  - 文案颜色可自定义；为保证在深/浅桌面上可读，文字默认带描边或半透明底板。
  - 文案非必填；配置后悬停显示、移开消失；未配置则悬停不显示文案。

### 6.2 事项提醒分区
- **素材上传**：支持视频 / 动图；**仅 1 个**；支持自由缩放（最大全屏，最小快捷图标大小）。
- **提醒时间设置**：滚动选择 年-月-日-时-分。
- **提醒文案**：≤ 50 字，白底黑字面板展示。
- **提醒面板**：含文案 + "我知道了"按钮（见 §4.2）。
- 支持**重复规则**：单次 / 每天 / 每周 / 工作日。
- **提示音**：可开关；可选内置音效或上传自定义音频（决策 G）。

### 6.3 防沉迷分区
- **素材上传**：支持视频 / 动图；**仅 1 个**；尺寸固定为**全屏展示**。
- **触发设置**：设置"连续使用时长"（分钟）作为触发阈值，**默认 90 分钟**；设置"空闲多久重置计时"的空闲阈值，**默认 10 分钟**。
- **休息时长设置**：设置触发后经过多少分钟自动结束，恢复陪伴办公状态。

## 7. 文件 / 素材规范（重要技术约束）

> 格式策略（决策 F）：**不限制上传格式，也不做透明化/裁剪处理**，观感由用户自行负责。

- **支持格式**：常见视频（mp4、webm、mov 等）与动图（gif、apng）。
- ⚠️ **风险提示（须在上传界面告知用户）**：普通不透明视频（如普通 mp4）悬浮桌面会出现黑/白矩形边框，破坏"悬浮真实感"。**推荐用户优先使用透明通道素材**：APNG、透明 GIF、WebM(VP9 alpha)、HEVC alpha 的 MOV。程序不强制、不自动处理。
- **文件大小/规格限制**（默认值，可后续调整）：单文件 ≤ 50MB；分辨率建议 ≤ 1080p；超限时提示但不强制拦截。用于控制内存与 CPU 占用。
- 最小尺寸（快捷图标大小）下需保证可拖动、可双击的最小可点击区域。

## 8. 时间选择器规则

| 单位 | 取值范围 |
|---|---|
| 分 | **0–59** |
| 时 | **0–23** |
| 日 | **1–31，随所选年月动态变化**（处理小月与闰年 2 月） |
| 月 | 1–12 |
| 年 | 公元滚动，**需设定合理上下限**（如当前年 ～ 当前年+10） |

- **过去时间处理**：禁止选择已过去的时间点（或选择后即时提示无效），不允许设置后立即误触发。

## 9. 非功能性需求

- **性能**：视频常驻播放 + always-on-top，需控制 CPU/GPU/内存占用；多素材轮播时注意切换内存释放。
- **持久化**：所有配置（素材、时间、文案、位置、自启等）本地持久化，重启恢复。
- **开机自启**：可在控制面板开关。
- **稳定性**：长时间运行不内存泄漏；锁屏/解锁、显示器插拔等场景不崩溃。

## 10. 技术规格与实现要点（基于 Electron）

> 技术栈见决策 D。

### 10.1 架构概览（多窗口分离）

| 窗口 | 职责 | 关键属性 |
|---|---|---|
| 宠物窗 | 渲染动态形象、承载拖动/悬停/点击交互 | `transparent:true`、`frame:false`、`alwaysOnTop:true`、`skipTaskbar:true` |
| 控制面板窗 | 三分区配置 + 通用设置 | 常规窗口 |
| 防沉迷全屏窗 | 覆盖层，全屏休息素材 | 全屏、覆盖宠物所在屏 |
| 提醒面板窗 | 最高叠加层，叠加在任意状态之上 | 最高置顶层级 |

### 10.2 关键实现点
- **透明悬浮窗**：`BrowserWindow` 设 `transparent: true`、`frame: false`、`alwaysOnTop: true`、`skipTaskbar: true`；动态形象用 HTML `<video>`（webm alpha）或 `<img>`（APNG/GIF）渲染。
- **置顶层级**：`setAlwaysOnTop(true, 'screen-saver')` 提升到较高层级；普通窗口可覆盖，独占全屏游戏不强制穿透（决策 C）。
- **拖动**：透明窗内用 `-webkit-app-region: drag` 或自实现拖拽 + 移动阈值区分单击/拖动。
- **活动检测（防沉迷计时）**：用系统空闲时间 `powerMonitor.getSystemIdleTime()` 判断无操作时长，满阈值重置累计计时。
- **锁屏/睡眠**：监听 `powerMonitor` 的 `lock-screen` / `suspend` / `unlock-screen` / `resume` 事件，控制宠物休眠与计时暂停。
- **独占全屏检测**：通过原生模块/系统 API 判断前台是否独占全屏，决定是否抑制弹出。
- **多窗口结构**：宠物窗、控制面板窗、防沉迷全屏窗、提醒面板窗分离管理；提醒面板为最高置顶叠加层。
- **持久化**：配置与素材路径用本地存储（`electron-store` + 本地文件目录）。
- **开机自启**：`app.setLoginItemSettings`。

### 10.3 安全与隐私
- **全本地、无联网、无遥测**：不收集、不上传任何用户数据。
- 素材文件仅存本地用户目录；卸载时提示清理。
- 不申请非必要系统权限；活动检测仅读取系统空闲时长，不记录按键内容（非键盘监听器，仅 idle 时间）。

## 11. 风险与路线图

### 11.1 分阶段路线图（Phased Rollout）

**MVP（核心可用）**
- 透明悬浮宠物窗 + 拖动（含移动阈值）+ 位置记忆。
- 右键菜单（面板/暂停/退出）+ 双击开面板，防误退出。
- 陪伴办公基础态：单/多素材展示 + 悬停文案。
- 控制面板骨架 + 配置本地持久化 + 开机自启。
- 目标：K1、K3 达标。

**v1.1（三功能闭环）**
- 事项提醒：提醒面板、可堆叠列表、重复规则、提示音、时间选择器规则。
- 防沉迷：活动检测计时、阈值触发/空闲重置、全屏休息素材、休息时长自动结束。
- 图层叠加状态机：优先级、打断暂停/恢复、暂停宠物联动。
- 目标：K4、K5 达标。

**v2.0（健壮性与体验打磨）**
- 锁屏/睡眠/显示器插拔全场景鲁棒性；独占全屏检测与抑制弹出。
- 文案可读性（描边/半透明底板）、缩放体验、内存切换释放优化。
- 长稳测试。目标：K2、K6 达标。

**Backlog（后续按需）**：多显示器自选屏、更多内置素材/音效、素材市场。

### 11.2 技术风险登记表（Technical Risks）

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| R1 不透明素材黑/白边框破坏悬浮感 | 体验受损（决策 F 不处理） | 上传界面强提示 + 推荐透明格式 + 提供合规内置素材示例 |
| R2 视频常驻播放导致 CPU/GPU/内存偏高 | 影响 K1/K2 | 限制分辨率/大小、轮播切换释放资源、空闲降帧/暂停渲染策略 |
| R3 独占全屏检测在不同显卡/全屏模式下不准 | 防沉迷误弹出干扰游戏 | 原生 API 多策略判断 + 失败时保守抑制弹出 |
| R4 拖动阈值与单击/双击判定冲突 | 误触退出/误开面板（影响 K3） | 统一手势状态机 + 移动阈值 + 双击时间窗调参 |
| R5 防沉迷计时在锁屏/睡眠/打断下错乱 | 影响 K5 | 集中计时器 + 电源事件钩子 + 状态机用例全覆盖 |
| R6 always-on-top 被某些全屏/独占程序压制 | 提醒不可见 | 提升至 screen-saver 层级；独占全屏按决策 C 延迟处理 |
| R7 长时间运行内存泄漏 | 影响 K2 | 视频元素生命周期管理 + 72h 长稳 + 内存曲线监控 |
