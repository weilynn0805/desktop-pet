# 桌面宠物 实施计划（Implementation Plan）

> 配套文档：`PRD-桌面宠物-合并版.md`（v3.0）。本计划是把 PRD 拆解为可执行的工程任务，供编程 agent 按阶段实施。
> 技术栈：Electron + 原生 HTML/CSS/JS（前端不强制框架；控制面板可用轻量框架，详见 §1.3）。
> 平台：仅 Windows 桌面。纯本地、无联网。

---

## 0. 总体原则（给编程 agent 的约束）

1. **严格按 PRD 决策实现**：A–G 决策（防沉迷连续时长触发/非强制/独占全屏不穿透/Electron/默认90+10分/不处理素材格式/内置+自定义音效）不得擅自更改。
2. **多窗口分离架构**（PRD §10.1）：宠物窗、控制面板窗、防沉迷全屏窗、提醒面板窗，各自独立 `BrowserWindow`，主进程统一调度。
3. **状态机集中在主进程**：图层优先级、计时、打断/恢复逻辑全部由主进程的状态机与计时管理器掌管，渲染进程只负责展示与上报交互（避免计时归属混乱）。
4. **进程间通信用 IPC**：渲染进程不直接访问 Node/系统能力；通过 `contextBridge` + `ipcRenderer` 暴露受控 API（`contextIsolation: true`、`nodeIntegration: false`）。
5. **分阶段交付**：按 PRD §11.1 路线图 MVP → v1.1 → v2.0 推进，每阶段对应 KPI 验收。
6. **不引入联网/遥测/账号**（PRD §10.3、Non-Goals）。

## 1. 项目脚手架与基础设施（阶段 0，先行）

### 1.1 工程初始化
- 初始化 `package.json`，依赖：`electron`、`electron-builder`（打包）、`electron-store`（持久化）。
- 目录结构建议：
  ```
  /src
    /main            主进程
      main.js              入口：app 生命周期、窗口创建
      windows/             各窗口创建与管理（pet, panel, fatigue, reminder）
      state/               状态机 + 计时管理器
      services/            store(持久化)、power(电源/锁屏)、activity(空闲检测)、
                           fullscreen(独占全屏检测)、autostart、audio、assets
      ipc/                 IPC 通道注册与 handler
    /renderer
      /pet             宠物窗页面 + 交互(手势状态机)
      /panel           控制面板页面（三分区 + 通用项）
      /fatigue         防沉迷全屏页面
      /reminder        提醒面板页面（可堆叠列表）
      /shared          公共样式/工具
    /preload           各窗口 preload（contextBridge 暴露 API）
  /assets              内置素材 + 内置音效
  /build               图标/打包资源
  ```
- 配置 `electron-builder`（Windows NSIS 安装包，后置阶段）。

### 1.2 配置与数据模型（核心，先定 schema）
用 `electron-store` 定义统一配置 schema（其他模块全部依赖它）：
```jsonc
{
  "pet": {
    "position": { "x": 100, "y": 100 },     // 位置记忆
    "scale": 1.0,                            // 缩放（最大全屏，最小快捷图标）
    "assets": [ { "id", "path", "type", "caption", "captionColor" } ], // 多素材+一对一文案
    "rotateMinutes": 10                      // 多素材轮播 5–60，单素材忽略
  },
  "reminders": [
    { "id", "assetPath", "text", "datetime", "repeat", // single|daily|weekly|workday
      "sound": { "enabled", "type": "builtin|custom", "src" }, "enabled" }
  ],
  "fatigue": {
    "asset": { "path", "type" },
    "triggerMinutes": 90,    // 决策 E
    "idleResetMinutes": 10,  // 决策 E
    "restMinutes": 5,        // 休息时长
    "sound": { ... }
  },
  "general": {
    "autoLaunch": false,
    "soundEnabled": true,
    "paused": false
  }
}
```
- 素材文件复制到用户数据目录（`app.getPath('userData')/assets`），store 只存相对路径，避免源文件移动后失效。
- 提供配置读写的封装服务 + 版本号字段（便于后续迁移）。

### 1.3 渲染层技术选型
- 宠物窗、防沉迷窗、提醒窗：**原生 HTML/CSS/JS**（轻量、低开销，利于 K1 资源占用）。
- 控制面板窗：表单较复杂，可用原生或轻量框架（如 Vue/Preact），由实现者决定，但需保持构建简单。

## 2. 宠物窗与交互（MVP 核心）

### 2.1 透明悬浮窗（services/windows/pet）
- `BrowserWindow`：`transparent:true`、`frame:false`、`alwaysOnTop:true`、`skipTaskbar:true`、`resizable:false`、`hasShadow:false`。
- 置顶层级：`setAlwaysOnTop(true, 'screen-saver')`（PRD §10.2）。
- 启动时从 store 恢复位置与所在显示器；窗口大小随缩放变化。
- **可点击区域**：透明区域需 `setIgnoreMouseEvents` 处理或用素材实际像素区域承载点击（最小尺寸下保证可拖动/可双击，PRD §7）。

### 2.2 素材渲染（renderer/pet）
- 视频用 `<video autoplay loop muted playsinline>`；动图用 `<img>`（APNG/GIF）。
- 透明通道：WebM(VP9 alpha) / APNG / 透明 GIF / HEVC alpha MOV。**不做任何转码/裁剪/透明化**（决策 F）。
- 缩放：CSS transform/尺寸控制，范围 = [快捷图标大小, 全屏]。

### 2.3 手势状态机（关键，对应 K3 / R4）
在渲染进程实现统一手势识别，区分：
- **单击**：按下→松开位移 < 移动阈值（如 5px）且单击 → 触发互动反应。
- **拖动**：按下后位移超过移动阈值 → 进入拖动，实时移动窗口（自实现拖拽，IPC 通知主进程 `moveWindow`，或 `-webkit-app-region: drag` 二选一，推荐自实现以精确控阈值）。松手后持久化位置。
- **双击**：两次单击在时间窗内（如 300ms）→ 打开控制面板。
- **右键**：弹出右键菜单（主进程 `Menu`）：打开控制面板 / 暂停宠物 / 退出。
- **悬停 hover**：陪伴办公态下若该素材绑定文案则头顶显示文案，移开消失（§4.1）。
- ⚠️ 退出**仅**经右键菜单"退出"，单击绝不退出（US-6 防误操作）。

### 2.4 头顶文案显示
- ≤20 字；颜色可配；默认带描边或半透明底板保证深/浅桌面可读（PRD §6.1，v2.0 打磨可读性，但 MVP 先实现基本显示）。
- 文案与素材一对一绑定；未配置则不显示。

### 2.5 多素材轮播
- 多素材时按 `rotateMinutes`（5–60）切换；单素材不轮播。
- 切换时释放上一个 video/img 资源（防内存泄漏，R7）。

## 3. 控制面板（MVP 骨架 → 逐步填充）

### 3.1 窗口与打开方式
- 常规 `BrowserWindow`；双击宠物或右键菜单"打开面板"打开；单例（已开则聚焦）。

### 3.2 三分区 + 通用项（PRD §6）
- **陪伴办公分区**：多素材上传/删除/排序、缩放设置、轮播时间（多素材才可设，5–60min）、每素材文案+颜色。
- **事项提醒分区**：单素材上传、提醒列表 CRUD、时间选择器（§8 规则）、重复规则、提醒文案(≤50字)、提示音开关+内置/自定义。
- **防沉迷分区**：单素材上传（全屏）、触发时长(默认90)、空闲重置(默认10)、休息时长。
- **通用项**：开机自启开关、提示音总开关、当前状态显示、暂停/恢复宠物。
- 所有改动经 IPC 写入 store，主进程热应用（无需重启）。

### 3.3 素材上传交互
- 文件选择对话框；展示 R1 风险提示（推荐透明格式）；大小>50MB / 分辨率>1080p 时提示但不拦截（决策 F、PRD §7）。

## 4. 状态机与计时管理器（主进程核心，v1.1）

### 4.1 图层状态机（PRD §3）
- 三层：基础层(陪伴办公) / 防沉迷覆盖层 / 提醒最高叠加层。
- 优先级：**事项提醒 > 防沉迷 > 陪伴办公**。
- 规则实现（§3.3）：
  1. 防沉迷活动时提醒到点 → 提醒面板叠加在防沉迷窗之上（更高置顶层级）。
  2. 提醒打断期间防沉迷倒计时**暂停**，用户确认后**继续**。
  3. 提醒不点不消失；多条到点 → 可堆叠列表逐条确认。
  4. 防沉迷可随时手动关闭/退出（决策 B），不阻止关闭。
  5. 暂停宠物 → 提醒与防沉迷计时一并暂停（§12）。
- 用显式状态机（如状态枚举 + 事件转移表）实现，避免布尔标志散落。

### 4.2 计时管理器（集中，对应 K5 / R5）
- **防沉迷累计计时**：基于活动检测；满 `triggerMinutes` 触发。
- **空闲重置**：`powerMonitor.getSystemIdleTime()` 轮询；空闲满 `idleResetMinutes` → 重置累计。
- **休息倒计时**：触发后计 `restMinutes`，到点自动结束回基础层；被提醒打断时暂停/恢复。
- **锁屏/睡眠**：监听 `powerMonitor` 的 `lock-screen`/`suspend` → 暂停并按规则重置；`unlock-screen`/`resume` → 恢复。
- 所有计时集中一处管理，统一 tick，便于暂停/恢复/调试。

## 5. 事项提醒（v1.1）

### 5.1 调度
- 主进程维护提醒调度：基于 store 中提醒列表计算下次触发时间（含重复规则展开：单次/每天/每周/工作日）。
- 到点误差 ≤ 1s（K4）：用计时器对齐，避免 `setInterval` 漂移（计算到下次触发的精确延迟）。
- 重复规则触发后自动计算下一次；单次触发后标记完成。

### 5.2 提醒面板窗（renderer/reminder）
- 最高置顶层级，叠加在任意状态之上（不替换底层）。
- 白底黑字文案 + "我知道了"按钮；**不点不消失**。
- 多条同时/先后到点 → 可堆叠列表，每条独立确认。
- 到点时宠物切换为提醒素材；全部确认后宠物恢复底层状态（防沉迷或陪伴办公）。

### 5.3 提示音（决策 G）
- audio 服务：内置音效（随包） + 用户上传自定义音频；受总开关与单条开关控制。

### 5.4 时间选择器（PRD §8）
- 滚动选择 年-月-日-时-分。
- 分 0–59、时 0–23、月 1–12、日 1–31 随年月动态（小月/闰年 2 月）、年 = 当前年～当前年+10。
- 禁止选过去时间（或选后即时提示无效）。

---

## 6. 防沉迷（v1.1）

### 6.1 全屏窗（renderer/fatigue）
- 触发时创建全屏窗，覆盖宠物所在屏（PRD §5.2）；展示防沉迷素材（全屏）。
- 善意提醒：可随时手动关闭/退出（决策 B），界面提供关闭入口。
- 达 `restMinutes` 自动结束，回陪伴办公态。

### 6.2 活动检测
- `powerMonitor.getSystemIdleTime()` 读取系统空闲时长（**不记录按键内容**，PRD §10.3）。
- 与计时管理器（§4.2）联动完成触发/重置。

## 7. 系统适配与健壮性（v2.0）

- **独占全屏检测**（决策 C / R3）：前台独占全屏程序时，防沉迷与提醒延迟/不弹出，退出全屏后再处理。Windows 下用原生 API（如 `QueryFullScreenWindow` 思路 / Shell API / 比较前台窗矩形与屏幕矩形）多策略判断；失败时**保守抑制弹出**。
- **锁屏/睡眠/唤醒**（K6 / R5）：宠物休眠（暂停动画与计时），恢复后正确还原。
- **显示器插拔**（K6）：监听 `screen` 的 `display-added/removed/metrics-changed`，校正宠物位置（防止落到不存在的屏）。
- **开机自启**：`app.setLoginItemSettings`，面板开关控制。
- **可读性打磨**：文案描边/半透明底板；缩放体验。
- **内存优化**（K1/K2/R7）：素材切换释放、空闲降帧/暂停渲染策略、video 元素生命周期管理。

## 8. IPC 通道清单（主↔渲染）

- `config:get` / `config:set` / `config:onChange`（配置读写与广播）
- `pet:move` / `pet:savePosition` / `pet:setScale`
- `pet:interact`（单击/悬停互动上报）
- `menu:show`（右键菜单）→ 返回动作：openPanel / pause / quit
- `panel:open`
- `pause:toggle`（暂停/恢复，联动计时）
- `assets:upload` / `assets:list` / `assets:delete`
- `reminder:add/update/delete/list` / `reminder:fired` / `reminder:ack`
- `fatigue:trigger` / `fatigue:close` / `fatigue:tick`
- `state:current`（当前状态查询/订阅）
- `audio:play`
- 全部经 preload `contextBridge` 暴露受控方法，参数校验。

## 9. 分阶段交付与验收（对齐 PRD §11.1 + KPI）

### MVP（阶段 1）
- 透明悬浮宠物窗 + 手势状态机（拖动阈值/单击/双击/右键/悬停）+ 位置记忆。
- 右键菜单（面板/暂停/退出）+ 防误退出。
- 陪伴办公：单/多素材展示 + 轮播 + 悬停文案。
- 控制面板骨架 + 配置持久化 + 开机自启。
- **验收**：K1（CPU≤5%/内存≤200MB）、K3（交互准确率100%）。

### v1.1（阶段 2）
- 事项提醒：面板/可堆叠列表/重复规则/提示音/时间选择器。
- 防沉迷：活动检测计时/阈值触发/空闲重置/全屏素材/休息自动结束。
- 图层叠加状态机：优先级/打断暂停恢复/暂停联动。
- **验收**：K4（提醒误差≤1s、重复规则100%）、K5（计时状态机100%）。

### v2.0（阶段 3）
- 锁屏/睡眠/显示器插拔鲁棒性；独占全屏检测与抑制。
- 文案可读性、缩放体验、内存切换释放优化。
- 72h 长稳测试。
- **验收**：K2（72h无崩溃、内存增长≤10%）、K6（场景鲁棒性）。

### 打包发布
- `electron-builder` 生成 Windows 安装包；卸载时提示清理素材目录（PRD §10.3）。

## 10. 测试要点（对应 KPI 验证方式）

- **手势用例集**：≥30 次拖动/单击混合，验证无误触退出（K3）。
- **提醒自动化时间用例**：到点误差、各重复规则正确性（K4）。
- **状态机用例**：覆盖 §4.3 全部规则——触发/空闲重置/打断暂停恢复/暂停联动（K5）。
- **场景回归**：锁屏解锁、睡眠唤醒、显示器插拔（K6）。
- **资源采样**：`process.getProcessMemoryInfo` + 任务管理器，1h 空闲态采样（K1）、72h 长稳内存曲线（K2）。

## 11. 给编程 agent 的实施顺序建议

1. 脚手架 + 配置 schema（§1）——一切的基础，先定 store 数据模型。
2. 宠物窗透明悬浮 + 手势状态机 + 位置记忆（§2）。
3. 右键菜单 + 控制面板骨架 + 持久化打通（§2.3、§3）。
4. 陪伴办公完整（多素材/轮播/悬停文案）→ **MVP 验收**。
5. 状态机 + 计时管理器骨架（§4）。
6. 事项提醒全链路（§5）。
7. 防沉迷全链路（§6）→ **v1.1 验收**。
8. 系统适配与健壮性（§7）→ **v2.0 验收**。
9. 打包发布 + 测试用例（§9、§10）。
