# MVP 计划 · 悬浮宠物底座（Floating Pet Core）

> 最小可行版本。目标：用一个能独立运行、肉眼可验证的最小闭环，验证桌宠的"底座"是否成立。
> 这是后续所有功能（陪伴文案、提醒、防沉迷、控制面板）的承载窗口——底座不稳，后面都白搭。

---

## 1. 为什么这是"最核心环节"

桌面宠物区别于普通应用的本质，就两点：**悬浮在所有窗口之上**、**像贴在桌面上一样无边框透明**。再加上桌宠必须能**被拖到任意位置并记住**、且**不会被误操作关掉**。这四件事是 Electron 桌宠最容易翻车的地方（黑边框 / 拖动闪烁抖动 / 单击拖动混淆 / 误退出），所以必须最先打通、最先验证。

本 MVP **只做这四件事 + 一个最小退出口**，其余全部砍掉。

---

## 2. 范围界定

### ✅ 本期做（IN）
1. 透明、无边框、置顶、不在任务栏的悬浮宠物窗。
2. 一个会呼吸/弹跳的占位宠物（纯 CSS 画，无需准备素材即可验证透明效果）。
3. 左键拖动移动 + **移动阈值**区分单击与拖动。
4. 位置记忆：拖动结束持久化，重启恢复到原位。
5. 右键菜单"退出"（防误退出：单击绝不退出）。
6. 单击互动反应（挤压动画）——顺带验证"单击 vs 拖动"判定是否准确。

### ❌ 本期不做（OUT，留给后续小步）
- 控制面板、双击开面板（无面板可开，先留 TODO）。
- 真实视频/动图素材、缩放、多素材轮播、悬停文案。
- 事项提醒、防沉迷、状态机、计时。
- 开机自启、提示音、独占全屏检测、锁屏处理。
- 打包安装包（先用 `npm start` 验证；打包放到功能稳定后）。

## 3. 原型与 UI 说明

- **形态**：屏幕右下角一只黄色圆球宠物，带眼睛和腮红，持续轻微上下弹跳（证明动画在跑、窗口透明无黑框）。
- **光标反馈**：悬停为 `grab`（小手），拖动时变 `grabbing`。
- **拖动**：按住左键拖动，宠物跟随鼠标平滑移动；松手停在原地。
- **单击**：轻点一下宠物，做一次"挤一挤"的回弹反应。
- **右键**：在宠物上右键 → 弹出仅含"退出"的菜单。
- UI 极简，无任何窗口边框/标题栏/任务栏图标——看上去就是"桌面上趴着一只宠物"。

## 4. 代码结构与职责

```
package.json                         # 仅依赖 electron，npm start 启动
src/
  main/
    main.js                          # 主进程：建透明窗、IPC(移动/存位置/菜单)、退出
    store.js                         # 极简 JSON 持久化(userData/config.json)
  preload/
    pet-preload.js                   # contextBridge 暴露受控 petAPI
  renderer/pet/
    index.html                       # 宠物窗页面（CSS 占位宠物）
    pet.css                          # 透明背景 + 弹跳/挤压动画 + 光标
    pet.js                           # 手势状态机(单击/拖动) + 右键菜单
```

关键技术点（已在代码注释标出）：
- 透明悬浮：`transparent/frame:false/alwaysOnTop/skipTaskbar` + `setAlwaysOnTop('screen-saver')` + CSS `background:transparent`。
- 安全隔离：`contextIsolation:true`、`nodeIntegration:false`，渲染进程只能用 preload 暴露的 `petAPI`。
- 拖动不丢帧：`setPointerCapture` 让指针移出窗口仍能收到 `pointermove`。
- 无累积漂移：每次用"按下时窗口位置 + 总位移"计算新位置，而非逐帧累加。
- 单击/拖动区分：位移 `> 5px` 才判定拖动，否则判定单击。

## 5. 运行方式

```powershell
# 在项目根目录 D:\claude\桌面宠物 下执行
npm install      # 首次：下载 electron
npm start        # 启动桌宠
```
退出方式：宠物上右键 →「退出」（开发时也可在终端 Ctrl+C）。

## 6. 测试用例（手动验收清单）

> 对应 PRD KPI：K3（交互准确率 100%、无误触退出）、K1（资源占用）。

| # | 用例 | 操作 | 预期结果 |
|---|---|---|---|
| T1 | 透明无边框 | 启动程序 | 宠物悬浮桌面，**无黑/白矩形边框**、无标题栏、任务栏无图标 |
| T2 | 置顶 | 打开浏览器/记事本等普通窗口并最大化 | 宠物始终浮在上层可见 |
| T3 | 动画 | 观察 | 宠物持续轻微弹跳，不卡顿 |
| T4 | 单击不误退出 | 在宠物上单击 ≥10 次 | 每次只做挤压反应，**程序绝不退出**、不移动 |
| T5 | 拖动移动 | 按住左键拖到屏幕各处 | 平滑跟随、不闪烁/不抖动，松手停稳 |
| T6 | 阈值区分 | 极小幅度按下松开(<5px) | 判定为单击(挤压)，**不发生位移** |
| T7 | 位置记忆 | 拖到左上角 → 右键退出 → 重新 `npm start` | 重启后宠物回到**左上角原位** |
| T8 | 防误退出 | 反复单击、双击、左键 | 均不退出；仅右键「退出」才关 |
| T9 | 右键退出 | 宠物上右键 →「退出」 | 程序正常关闭，进程结束 |
| T10 | 资源占用(K1) | 空闲态运行，看任务管理器 | CPU 平均 ≤ 5%、内存 ≤ 200MB |

混合操作专项（K3）：连续做 ≥30 次"拖动 + 单击"混合操作，统计是否有一次误判（该拖的没拖动 / 该单击的发生了位移 / 误退出），目标 **0 误判**。

---

## 7. 验收标准（Definition of Done）

- T1–T9 **全部通过**；T10 资源占用达标。
- K3 混合操作专项 0 误判。
- 代码可被你逐文件读懂：主进程/preload/渲染三层职责清晰，无隐藏依赖（除 electron 外零三方库）。
- 你能清楚回答："拖动是怎么实现的""位置存在哪个文件""为什么单击不会退出"——达到"掌握代码全貌"的目标。

---

## 8. 上线（本阶段 = 本地可用）

- 本 MVP 的"上线"= 在你自己机器上 `npm start` 稳定运行、通过第 6/7 节验收。
- **暂不打安装包**：避免过早引入 `electron-builder` 复杂度，等核心功能成形后再做（届时一条 `electron-builder` 配置产出 Windows 安装包，并在卸载时提示清理素材目录，见 PRD §10.3）。

---

## 9. 通过验收后的下一小步（建议顺序）

1. **把占位宠物换成真实素材**：`<img>`(APNG/GIF) 或 `<video>`(webm alpha)，验证真实素材的透明观感与资源占用。
2. **缩放**：右键菜单加放大/缩小，或滚轮缩放，验证 [快捷图标, 全屏] 范围。
3. **控制面板骨架**：双击宠物 / 右键「打开面板」开一个普通窗口，打通"面板改配置 → 主进程热应用 → 持久化"这条链路。
4. 再依次进入：悬停文案 → 多素材轮播 → 事项提醒 → 防沉迷。

每一步都保持"一个小功能 → 验证 → 再下一步"的节奏。
